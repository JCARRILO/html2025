Clase 8 · Imágenes responsivas en HTML

Práctica con srcset, sizes, picture, densidad 1x/2x, loading="lazy" y ajuste visual con object-fit.

1) srcset + sizes

El navegador elige la mejor imagen según el ancho mostrado.

2) <picture> (art direction)

Cambia la composición según el medio. En móvil cargamos un recorte estrecho; en escritorio, la imagen completa.

2b) <picture> por orientación

Usa consultas de medios para cambiar la imagen si el dispositivo está en vertical u horizontal.

2c) <picture> con densidad (1x/2x)

Para un ancho visual fijo, ofrece fuentes 1x y 2x dentro de picture.

2d) <picture> con prefers-color-scheme

Selecciona una imagen distinta para modo claro/oscuro. Demostración con activos existentes.

2e) <picture> con breakpoints

Cambia fuente según el ancho del viewport, con un fallback seguro.

2f) <picture> con diferentes pesos (hero)

Implementación como en la imagen: coloca hero-400.jpg, hero-800.jpg y hero-1200.jpg dentro de CLASE 8/8- IMG RESPONSIVE/IMG/.

Si aún no tienes esos archivos, usa la demo funcional siguiente que reutiliza imágenes de la Clase 3.

3) Densidad de píxeles (1x/2x)

Para elementos que muestran un ancho fijo en CSS, ofrece 1x/2x.

4) Lazy-loading

Carga diferida con loading="lazy". Se recomienda reservar espacio; aquí usamos aspect-ratio en el contenedor.

5) object-fit + aspect-ratio

Comparativa de cover vs contain con el mismo archivo.

6) Buenas prácticas rápidas

  • Siempre incluye alt descriptivo.
  • Usa srcset + sizes o picture según el caso.
  • Para evitar CLS, reserva espacio (atributos width/height o aspect-ratio).
  • Activa loading="lazy" en imágenes por debajo del primer pliegue.
  • Cuando puedas, sirve formatos modernos (WebP/AVIF) con <picture> y fallback a PNG/JPG.

2g) Ejemplo final: diferentes pesos reales

Usa imágenes existentes del curso para cargar distintas variantes según el ancho.

7) Ejemplos estilo screenshot

Mismo patrón del ejemplo: <picture> con breakpoints y un <img> con srcset + sizes, usando imágenes reales del curso.

8) Atributos loading y decoding

Controla cuándo se carga una imagen y cómo se decodifica antes de pintarse.
