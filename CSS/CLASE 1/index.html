¿Qué es CSS y cómo funciona la cascada?

CSS significa Cascading Style Sheets (hojas de estilo en cascada). La cascada implica que, cuando varias reglas afectan al mismo elemento, la regla más abajo (definida después) prevalece si la especificidad empata.

  • Los estilos posteriores sobrescriben a los anteriores si apuntan a lo mismo.
  • Planifica el orden y el alcance de tus reglas para evitar conflictos.

¿Cuáles son las tres formas de agregar estilos?

  1. Inline con el atributo style en la etiqueta.
  2. En el documento con <style> dentro de <head>.
  3. Hoja externa enlazada con <link rel="stylesheet" href="styles.css">.

Recomendación práctica: trabajar con archivo externo para mantenimiento, versionado y consistencia.

1) ¿Cómo se usa el atributo style en línea?

Agrega el estilo directamente a la etiqueta. Es inmediato, pero poco mantenible en proyectos grandes.

Texto con color en línea (inline).

<p style="color: blue;">
Texto con color en línea.
</p>
  • Ventaja: cambio puntual rápido.
  • Desventaja: difícil de mantener y reutilizar.

2) ¿Cómo funciona la etiqueta style en el head?

Define reglas en el documento. Es útil para demos o páginas pequeñas, pero escala mal en equipos y proyectos grandes.

<head>
  <style>
    h1 { color: blue; }
    h2 { color: green; }
  </style>
</head>

Este bloque de <style> en la cabecera de esta página pinta este título de sección.

3) ¿Por qué conviene una hoja de estilos externa?

Crea un archivo (por ejemplo styles.css) y enlázalo con <link>. Es la buena práctica para mantener, versionar y escalar estilos.

<head>
  <link rel="stylesheet" href="styles.css" />
</head>

/* styles.css */
h1 { color: blue; }
h2 { color: green; }
  • Ventaja: separación clara entre estructura y presentación.
  • Ventaja: reutilización y cambios globales simples.

¿Qué efectos verás al aplicar estilos por etiqueta?

Si seleccionas una etiqueta (por ejemplo, p), la regla afecta a todas las etiquetas p del documento. Sin embargo, un estilo inline específico puede prevalecer por cascada y especificidad.

Este párrafo queda en azul por estilo inline.

Este otro usa el color definido para <p> en la hoja externa.

/* styles.css */
p { color: #5795D3; }

<p style="color: blue;">Inline gana por especificidad</p>
<p>Usa el color de la hoja externa (#5795D3)</p>
  • Seleccionar por etiqueta aplica cambios a múltiples elementos.
  • Un estilo inline puede sobrescribir la regla general de la hoja externa.
  • La especificidad y el orden importan para evitar conflictos.
